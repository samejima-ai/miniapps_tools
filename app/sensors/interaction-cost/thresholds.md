# Interaction Cost 閾値

第 3 層 Interaction Cost 測定の UX 代理指標。L0 対話 §2.5 UX 3問プロトコルから生成。

---

## Must 閾値（critical Priority）

| # | 指標 | 閾値 | 関連機能 |
|---|---|---|---|
| IC-1 | 持出登録の操作ステップ数 | **≤ 3 タップ**（入力 → 確認 → 確定） | F2 CaaF入力 |
| IC-2 | CaaF 応答時間（LLM 抽出） | **p95 ≤ 5,000 ms** | F2 |
| IC-3 | 一覧初期表示時間 | **p95 ≤ 2,000 ms** | F3 一覧 |
| IC-4 | フィルタ / 検索の即時反映 | **≤ 300 ms** | F3 |
| IC-5 | 一括返却の完了時間（N≤30 想定） | **≤ 2,000 ms** | F4 クイック返却 |
| IC-6 | 一覧反映（INSERT 後） | **≤ 1,000 ms** | F2 / F4 |

## 参考類似サービス

- **LINE 自然文運用**（既存）: 入力速度の参考。「ノリ」を維持するため、入力体験は LINE と同等以下のステップ数を維持

---

## 測定方法

### Playwright 経由の機械測定（IC-1, IC-3, IC-4, IC-5）
- `tests/e2e/interaction-cost.spec.ts` で各シナリオの実時間を測定
- 結果を `delivery/interaction-cost-report.json` に出力
- CI で閾値超過時 fail

### Real User Monitoring（IC-2, IC-6、本番運用後）
- LLM 応答時間 / 一覧反映時間は本番 RUM で p95 を集計
- 月次レポートを `history/CHANGELOG.md` に追記

## 業界標準値との対比

| 項目 | 業界標準（spec デフォルト） | 本プロジェクト |
|---|---|---|
| クリック数 | 3-5 回 | 3 タップ（IC-1） |
| 遷移深度 | 3 以内 | Gate→Tab→Card で 3 |
| 応答時間 | 30 秒以内 | **p95 5秒**（業務速度のため厳格化、IC-2） |
| 完了率 | 95% 以上 | 同左 |
| エラー率 | 5% 以下 | 同左 |
