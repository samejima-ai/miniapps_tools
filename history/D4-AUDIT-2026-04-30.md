### 3.2 MEDIUM（運用ルール文書化のギャップ）

#### M-1: `CHANGELOG.md` 全バージョンが `(in progress)` のままで完了マーク漏れ

**症状**:

- REGIME-LOG.md / INTENT.md / ARCH-DECISIONS.md は `## v5.x.x` 形式で明示せず該当しない

- 動作には影響なし

**改修候補**: minor PR で `(in progress)` → `(released 2026-04-XX)` へ更新（patch 寄りだが minor で吸収可能）

#### M-2: S/U/R 用語の対応関係が単一箇所で宣言されていない

**症状**:
- `S/U/R` 表記: 30 箇所（複数 SKILL / references / DIMENSIONS / INSIGHTS）
- `Scale / Uncertainty / Risk` 表記: 2 箇所（DH-PHILOSOPHY-INSIGHTS.md のみ）
- `規模・不確実性・リスク` 表記: 5 箇所（layer0-spec-architect SKILL.md / regime-assessment.md / subphase-selection.md）

**分析**:
- 三者は同義（regime-assessment.md §3 で「規模（S）」のように個別対応はあり）
- ただし「S = 規模 = Scale」「U = 不確実性 = Uncertainty」「R = リスク = Risk」を**単一箇所で宣言**する規格が不在
- `harness-verifier/glossary.yml` は dimensions / pillars は定義するが S/U/R 用語は未定義

**重要度判定**: **MEDIUM**
- 用語揺れは現状で誤解を生むレベルではない
- ただし新規 D4 改修（v5.5.0 の S/U/R 統合実装等）の際に混乱の温床

**改修候補**: minor PR で `harness-verifier/glossary.yml` に `score_axes` キーを新設し、対応表を宣言。あるいは `regime-assessment.md` 冒頭で対応表を明示。
