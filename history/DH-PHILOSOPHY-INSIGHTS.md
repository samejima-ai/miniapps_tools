### 4.4 Council の判断閾値（S/U/R 四段階）

§4.1〜§4.3（後続ハンドオフで追記予定）では Council と人間の権限分離を扱う想定であり、
本節はそれを前提に Council 内部での判断機構を扱う。

Council はサイクル中のあらゆる判断モーメントを Hook するが、
全てを同じ重さで扱うわけではない。三軸スコアリング（Scale / Uncertainty / Risk）により、
判断は以下の四段階に分類される：

| S/U/R 状態 | 判断主体 | 介入タイミング |
|---|---|---|
| 全て低 | L1 自律完結 | Council 介入なし |
| いずれか中 | L1 + Council | 事後献上 |
| いずれか高 | L1 + Council | 事前献上 |
| 複数高 | L1 + Council + 人間（D5） | 上申 |

この四段階閾値により、判断の重要度に応じた段階的な介入が実現される。
重要な判断のみが人間（D5）に到達するため、
「人間が必要な時にだけ介入する状態」が運用上担保される
（§4.3 で本主旨を扱う想定、後続ハンドオフで追記予定）。

Council の貫通性（停止判断含むあらゆる判断モーメントへの Hook）の
詳細は第 10 章を参照。

## 第 10 章: 自律サイクルと Council の貫通性

### 10.1 章の射程と 5 次元論との接続

本章は **D1〜D3 の射程**を扱う。D4 不変性は侵さない。

5 次元論の運用上の基準自体は `history/DIMENSIONS.md` を参照しつつ、
本ファイル内の第 2 章および §9.1〜9.3 には後続ハンドオフで対応関係を追記する前提とする。
したがって本章は、D4（設計層）の原則を前提にしつつ、
その下位層である D1〜D3 における**駆動メカニズムと判断機構**を扱う。

業界最先端の PR 完遂ループや ACI といった概念群は、本質的には D1〜D3 の話であり、
これらを D4 の議論と取り違えると、「DH の完遂概念を PR マージで上書きする」
といった誤った判断に至る。本章はこの取り違えを構造的に防ぐための章である。

### 10.2 自律サイクルの三層駆動構造

DH の自律駆動を支える駆動構造は、以下の三層で構成される：

| 層 | 役割 | 対応概念 |
|---|---|---|
| 抽象的生存条件 | サイクル回転による生存定義 | P2 情報代謝 |
| 物理的駆動装置 | サイクルを物理的に回す機構 | PR 完遂ループ（Cell 層） |
| 判断吸収機構 | サイクル中の判断モーメントを吸収 | Council System |

この三層は**独立した機構ではなく、垂直に積み上がる依存関係**を持つ：

- 上位層（生存条件）が無ければ、下位層（駆動装置）はただの作業ループに退化する
- 下位層（駆動装置）が無ければ、上位層（生存条件）は抽象的な題目に留まる
- 判断吸収機構が無ければ、駆動装置は判断ボトルネックで停止する

業界最先端の PR 完遂ループは**第二層のみを精緻化**しており、
第一層（なぜループが必要か）と第三層（判断委譲の構造）を持たない。
これが DH との最大の構造的差異。

### 10.3 PR 完遂ループの次元別扱い

PR 完遂ループは、5 次元論における**どの次元で語るか**で扱いが完全に変わる：

| 5 次元 | PR 完遂ループの扱い |
|---|---|
| D1（自律駆動の最終成果物） | **駆動装置として内包**。Cell 層の代謝サイクル一回転 |
| D2（D1 を生み出す足場） | PR 完遂ループを支援する ACI / SK / CI 環境を整備 |
| D3（配布される skill） | PR 完遂ループを駆動する SK を利用者プロジェクトに配布 |
| D4（設計層） | **完遂概念を上書きしない**。PR 完遂は D1 の駆動装置に過ぎない |
| D5（最終監督者） | 不変。PR 完遂ループは介在しない |

この扱いの違いは、§8.2（後続ハンドオフで追記予定）で扱う「PR マージを完遂とする世界観の拒否」と
§8.1 項目 4 の「PR 完遂ループパターンの内包」を**論理的に整合させる鍵**である。

両者は矛盾しない。D4 で拒否し、D1〜D3 で内包するからである。
DH のフラクタル構造ゆえに、同じ概念が次元によって異なる扱いを受けることは自然な帰結。

### 10.4 フラクタル粒度マッピング

DH の自律サイクルはフラクタル構造を持つため、サイクル粒度は階層ごとに異なる：

| 階層 | サイクルの粒度 | トリガー | 完了シグナル | 想定時間 |
|---|---|---|---|---|
| Cell | 1 Issue → 1 PR → マージ | Issue 起票 | PR マージ | 数時間〜数日 |
| Application | 1 機能要望 → N Issue → N PR → リリース | 機能要望 | リリース完了 | 数日〜数週 |
| Platform | 1 事業仮説 → N App → 検証完了 | 事業仮説 | 仮説の採否判断 | 数週〜数ヶ月 |
| 組織 | 1 ビジョン → N 事業 → 五段階収益モデル進行 | ビジョン定義 | 次段階移行 | 数ヶ月〜数年 |

各階層のサイクルは、**下位階層のサイクル群を内包**する形で構成される。
Cell 層の PR マージが N 回累積することで Application 層のリリースが達成され、
Application 層のリリースが N 回累積することで Platform 層の仮説検証が達成される。

業界最先端の議論は **Cell 層のみ**を対象としており、上位階層への自然な拡張機構を持たない。
DH はフラクタル原則 P1 により、同じ駆動構造が全階層で再帰する。

### 10.5 Council の判断閾値と貫通性

Council System は「上位の判断者」ではなく、**自律サイクルのあらゆる判断モーメントを貫通する判断レイヤー**である。

#### 10.5.1 判断閾値（三軸スコアリング S/U/R）

サイクル中の各判断モーメントは、三軸スコアリング（Scale / Uncertainty / Risk）により以下の四段階に分類される：

| S/U/R 状態 | 判断主体 | 介入タイミング |
|---|---|---|
| 全て低 | L1 自律完結 | Council 介入なし |
| いずれか中 | L1 + Council | 事後献上 |
| いずれか高 | L1 + Council | 事前献上 |
| 複数高 | L1 + Council + 人間（D5） | 上申 |

#### 10.5.2 貫通性の意味

Council は通常の事業判断・実装判断のみを扱うのではない。
**自律サイクルの停止判断すら Council が Hook して判定する**。

具体例：

- Cell 層で自己修復ループが発火した際、停止条件（試行回数閾値・進捗の質的変化検知・三軸スコアの再評価）も Council が Hook で判定する
- Application 層でリリース判定が必要な際、リリース可否を Council が Hook で判定する
- Platform 層で事業仮説の採否判断が必要な際、Council が Hook で判定する

これにより、DH の判断機構は単一化される：
**判断という判断は全て Council を通る**。
判断機構の冗長性が消え、責務範囲が明確になる。

#### 10.5.3 業界最先端との対比

業界最先端の構成では、判断は分散している：

- コンフリクト解消はエージェントの自律判断
- レビュー承認は別エージェント（Review Agent）の判断
- 最終マージ判断は人間
- システム全体の方針判断も人間

これは判断機構が**階層化されていない**ことを意味する。
判断の重要度に応じた段階的な介入機構が無く、結果として「人間が事後分析」という運用に陥る。

DH の Council は、判断の重要度を S/U/R で測定し、自律完結 / 事後献上 / 事前献上 / 上申に
段階的に振り分ける。これにより**重要な判断のみが人間に到達する**構造が実現される。

### 10.6 ACI の代謝駆動における役割

§8.1 で取り込みが確定した ACI（Agent-Computer Interface）概念は、
本章の文脈では**自律サイクルの物理層で機能する認知 IF 分離機構**として位置づけられる。

#### 10.6.1 ACI の本質

ACI は「観測者の認知特性ごとに、情報の見せ方を分離する」という思想。
人間用 UI（GUI / CLI）は人間の認知特性に最適化されているが、
LLM エージェントに同じ UI を使わせると、長大なログでコンテキストを食われ判断品質が劣化する。

ACI は AI エージェント専用に最適化された抽象化層として、以下を提供する：

- ファイルの該当行±数行だけを抽出して返す
- コマンド出力を要約・構造化して返す
- 致命的な破壊操作にガードレールを設ける

#### 10.6.2 ACI と DH の SK / RL / WF ハーネスの関係

DH の SK / RL / WF ハーネスは、ACI の上位実装として位置づけられる：

| 観点 | 一般的な ACI | DH の SK / RL / WF |
|---|---|---|
| 対象 | コマンド出力・ファイル表示の抽象化 | 行動様式そのもの（何をどう判断するか）も含む |
| 階層 | ツール層 | 行動文法層 |
| 駆動 | エージェントがツールとして呼ぶ | エージェントの認知体系として常駐 |

つまり DH の SK は「ACI を内包しつつ、さらに行動文法・判断制約まで含む上位概念」。

#### 10.6.3 P4 分離原則への内包

ACI の本質である「観測者の認知特性ごとに情報の見せ方を分離する」は、
P4「対話と実装の分離」の自然な拡張線上にある。

P4 の射程は、ACI 概念の取り込みにより以下のように拡張される：

- **役割分離**（既存）：人間は何を望むかを語り、AI は如何に実現するかを担う
- **認知 IF 分離**（拡張）：人間用 UI と AI 用 ACI は別物として設計する

両者は P4 内部の二段構造として統合され、別ピラー化はしない。

### 10.7 本章のまとめ

本章で確立された原則：

1. 自律サイクルは三層駆動構造で構成される（生存条件 / 駆動装置 / 判断吸収機構）
2. PR 完遂ループは D1〜D3 で内包し、D4 で拒否する（次元別扱い）
3. サイクル粒度は階層ごとに異なる（フラクタル粒度マッピング）
4. Council はあらゆる判断モーメントを貫通する判断レイヤー（停止判断含む）
5. ACI は P4 分離原則に内包される（認知 IF 分離として）

これらは全て D1〜D3 の射程に収まり、D4 不変性を侵さない。
業界最先端の技術的要素を、DH の哲学的座標系の中で**正しい次元に位置づけて取り込む**ための指針として機能する。
